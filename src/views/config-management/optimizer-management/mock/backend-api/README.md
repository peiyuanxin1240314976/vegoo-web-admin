# 优化师管理 · 接口清单

**父级路径**：`/api/config-management/optimizer`

| 方法 | 路径                 | 说明                                            | 优先级 |
| ---- | -------------------- | ----------------------------------------------- | ------ |
| POST | `/table`             | 分页列表（筛选、排序由服务端处理）              | P0     |
| POST | `/overview`          | 页头统计卡片（总数/在职/最低消耗均值/本月新增） | P0     |
| POST | `/meta-system-users` | 新建弹窗：可选系统用户列表                      | P0     |
| POST | `/`                  | 新增优化师                                      | P0     |
| PUT  | `/:id`               | 更新优化师                                      | P0     |
| POST | `/export`            | 导出                                            | P1     |

## 拆分原则

- 列表与 KPI **分开**：表格分页场景下不应仅靠当前页推算全量统计。
- 「关联用户」独立 **meta**，避免把整站用户表塞进列表接口。

详细字段见各 `*.json` 契约文件。
