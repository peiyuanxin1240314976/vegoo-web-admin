# 经营报告 — 后端联调核对清单

> 与 `mock/backend-api`、`src/api/business-report.ts`、`reportService.ts` 当前实现对齐。联调前双方按本节勾验。

## 1. 网关与路径

| 项 | 值 |
| --- | --- | --- | --- |
| 报告数据前缀 | `POST {ANALYSIS_API_BASE}/report` → 一般为 **`/api/v1/datacenter/analysis/report`** |
| 周期 | 在 **URL 路径** 段：`/daily | weekly | monthly/{feature}`，**不在** JSON body 里传 `period` |
| 飞书 | **`/api/v1/lark/*`**（与 report 前缀不同） |

## 2. 通用 JSON 业务体（除 app-list / 对比补充外）

所有 `overview`、`ad-platform`、`by-country`、`platform-country`、`campaigns` 的 POST body **必须**包含：

| 字段              | 类型       | 说明                                             |
| ----------------- | ---------- | ------------------------------------------------ |
| `startDate`       | `string`   | `YYYY-MM-DD`                                     |
| `endDate`         | `string`   | `YYYY-MM-DD`                                     |
| `appId`           | `string`   | 侧栏选中「整体」→ `""`；选中某应用 → 该应用 `id` |
| `appIds`          | `string[]` | 顶栏应用多选；**`[]` = 不限**                    |
| `platformList`    | `string[]` | 顶栏终端平台多选；**`[]` = 不限**                |
| `sourceList`      | `string[]` | 顶栏广告平台多选（枚举 string）；**`[]` = 不限** |
| `countryCodeList` | `string[]` | 顶栏国家多选（小写 alpha-2）；**`[]` = 不限**    |
| `account`         | `string`   | 广告账户；不限 → `""`                            |

**已废弃（前端不再发送）**：单笔 `platform`、`source`、`countryCode` 字符串。

## 3. `app-list` 专用

路径：`POST .../report/{period}/app-list`

### 3.1 字段（在 §2 基础上）

| 字段    | 类型     | 说明                                                                     |
| ------- | -------- | ------------------------------------------------------------------------ |
| `appId` | `string` | **固定 `""`**（与侧栏行选中无关）                                        |
| `tab`   | `string` | `summary` / `adPlatform` / `byCountry` / `platformCountry` / `campaigns` |

响应业务体：`{ "items": AppListItem[] }`（键名固定为 `items`，与契约一致）。

### 3.2 设计说明（避免误解为「只要 id 列表」）

1. **语义**：`app-list` 返回的是**当前报表上下文下**的侧栏数据（整体行 + 各应用卡片及卡片上指标），**不是**全站静态应用 id 列表。因此必须带与详情一致的 **`startDate` / `endDate`** 与顶栏四数组 **`appIds`、`platformList`、`sourceList`、`countryCodeList`**（空数组表示该维度不限）。
2. **`appId` 为何出现在 body 且必须为 `""`**：与详情接口共用同一套 POST 字段；**`""` 表示「列表模式」**，区别于详情里「非空 `appId` = 某一应用、空串 = 整体」。**顶栏应用范围只看顶栏 `appIds`，不要与详情 `appId` 混淆。**
3. **`account`**：与统一筛选维度对齐；**当前页无账户下拉，前端恒传 `""`**，后端可按「不限」忽略；未来若加顶栏账户筛选可同一字段生效。
4. **顶栏 `appIds` vs 详情里的 `appId`**：前者是顶栏**多选范围**；后者是侧栏**当前选中行**（仅详情请求使用）。两者同时有值时的组合规则（如交集）须在接口文档写死，见 §5。
5. **`app-list` 里的 `appId`/`account` 不是「旧代码兼容」**：与已删除的 `appList` 响应键、归一化补丁不同；二者属**统一 POST 契约**（见 README 同章「与已删除的…区别」）。

更完整的叙述见 **`mock/backend-api/README.md`** 中的「侧栏 `app-list` 接口设计说明（给后端）」。

## 4. 对比模式

路径：`POST .../report/{period}/compare-overview|compare-trends|compare-metrics`

在 §2 基础上增加：

| 字段               | 类型       | 说明                                          |
| ------------------ | ---------- | --------------------------------------------- |
| `compareAppIds`    | `string[]` | 对比勾选的应用 id（与顶栏范围 `appIds` 分列） |
| `compareEnabled`   | `boolean`  | 是否展示对比期                                |
| `compareStartDate` | `string`   | 对比期起                                      |
| `compareEndDate`   | `string`   | 对比期止                                      |

顶栏多选仍会随请求提交（与主报告一致）；对比区内日期变更由组件内部状态驱动。

## 5. 多选数组语义（须书面约定）

同一维度内多项：**OR 还是 AND**、与空数组「不限」的组合规则，**请后端在接口文档写死**，避免前后端理解不一致。

## 6. 响应与网关包装

- 前端 `unwrapReportPayload` 会向下剥最多 4 层 `data`；请确认网关与业务体层级与项目其它分析接口一致。
- 列表无数据时：`items` / 表格主数组建议 `[]`，避免 `null`。

## 7. 开关与本地 Mock

`src/views/product-insight/business-report/config/data-source.ts` 中 **`BUSINESS_REPORT_USE_MOCK`** 按枚举 **逐接口** 切 `false` 后走真实 HTTP；建议先 **`DailyAppList` + `DailyOverview`**，再扩到其余 Tab 与周月报。
