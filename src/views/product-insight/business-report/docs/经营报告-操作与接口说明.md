## 2. 页面结构（用户在页面上能做什么）

```
┌─────────────────────────────────────────────────────────────────┐
│ [日报] [周报] [月报]     对比模式开关   飞书推送   导出            │
├─────────────────────────────────────────────────────────────────┤
│ 筛选：应用 / 平台 /（随 Tab 变：广告平台、国家、系列状态等）        │
│ 日期：单日 / 自然周 / 自然月    对比昨日|上周|上月 开关              │
├─────────────────────────────────────────────────────────────────┤
│ [汇总] [广告平台] [分国家] [广告平台分国家] [在投广告系列]           │
├──────────────┬──────────────────────────────────────────────────┤
│ 侧栏：应用列表 │ 主区：当前周期 × Tab 对应的内容组件                │
│ （独立接口）   │ （overview + 当前 Tab 专属接口）                   │
└──────────────┴──────────────────────────────────────────────────┘
```

- **对比模式打开后**：隐藏侧栏 + 普通主区，改为全宽 **对比视图**（`MonthlyCompareMode.vue`，日报/周报/月报均使用该组件，随 `period` 切换对比接口）。
- **顶栏「应用 / 平台 / 广告平台 / 国家」选项**：来自 **驾驶舱公用接口**（见下文「其他依赖」），**不是**本模块 `mock/backend-api` 里的 JSON。

## 3. 普通模式数据流（核心逻辑）

### 3.1 两条独立链路

| 链路 | 作用 | 是否带 `appId`（单应用） |
| --- | --- | --- |
| **A. 侧栏应用列表** | 渲染左侧卡片，首项一般为「整体」 | **否**：`app-list` 请求体里 **`appId` 固定传 `""`**，并带 **`tab`** |
| **B. 右侧详情** | KPI、表格、图表等 | **是**：选中「整体」时前端不传/空；选中某应用时传该应用 `id` |

**给后端**：`app-list` 为何要带 **`appId: ""`**、**`account`**、以及日期与顶栏四数组（而不是「只返回应用 id 列表」），见 **`../mock/backend-api/README.md`** 中的 **「侧栏 `app-list` 接口设计说明（给后端）」**；联调核对摘要见 **`后端联调核对清单.md`** §3.2。

### 3.2 何时拉侧栏（链路 A）

在 **未开启对比模式** 时，当下面任一变化会触发 **`refreshSidebarAppList`** → 调用 **`getReportAppList`**：

- 周期：`daily` / `weekly` / `monthly`
- 二级 Tab：`summary` / `adPlatform` / `byCountry` / `platformCountry` / `campaigns`
- 报告日期：`reportDayYmd` / `reportWeekStartYmd` / `reportMonthYm`
- 顶栏筛选（与页面 **多选** 一致，请求体为 **数组**）：`filterAppIds`、`platformList`、`sourceList`、`countryCodeList`；**空数组 `[]` 表示该维度不限**

随后会 **串联** 一次 **`refreshReportDetail`**（见下）。

**仅切换二级 Tab** 时：会重拉侧栏（因 `tab` 参与 `app-list`），但 **尽量保留该 Tab 下已选应用**（与「改日期/周期」时默认回到整体不同）。

**切换周期 / 日期 / 上述筛选** 时：在侧栏返回且用户未在中间改选的前提下，会将当前 period+Tab 的选中 **重置为「整体」**，再拉详情。

**点击左侧某一应用卡片**：**只**走 **`refreshReportDetail`**，**不再**请求 `app-list`。

### 3.3 何时拉右侧详情（链路 B）

在 **未开启对比模式** 时触发 **`refreshReportDetail`**：

1. 每次 **`refreshSidebarAppList`** 成功并处理完选中态之后（必调一次）。
2. 用户 **点击侧栏应用**（与当前选中不同）时。

每次详情请求 **固定先调** **`getSummary`（overview）**，再根据当前 Tab **额外**调用：

| 当前二级 Tab                     | 除 overview 外再调用的接口 |
| -------------------------------- | -------------------------- |
| 汇总 `summary`                   | 无（仅 overview）          |
| 广告平台 `adPlatform`            | `ad-platform`              |
| 分国家 `byCountry`               | `by-country`               |
| 广告平台分国家 `platformCountry` | `platform-country`         |
| 在投广告系列 `campaigns`         | `campaigns`                |

### 3.4 对比模式与普通模式互斥

- **`compareMode === true`**：`refreshSidebarAppList` / `refreshReportDetail` **直接 return**，不拉普通报告接口。
- **关闭对比模式**：会 **`refreshSidebarAppList(true)`**（回到整体并拉侧栏 + 详情）。

若在详情请求 **进行中** 打开对比模式，返回后会 **丢弃** 本次普通详情写入（避免污染对比态）。

## 4. 对比模式数据流

组件：`components/MonthlyCompareMode.vue`。

在对比区内变更 **勾选应用、对比开关、对比区日期** 等时，会 **并行** 请求三个接口（周期随 `period` 替换 `daily|weekly|monthly`）：

1. `POST .../report/{period}/compare-overview`
2. `POST .../report/{period}/compare-trends`
3. `POST .../report/{period}/compare-metrics`

请求体在通用报告字段基础上增加：`appIds`、`compareEnabled`、`compareStartDate`、`compareEndDate`（见契约 `*-06/07/08`）。**通用顶栏多选**（`filterAppIds` 等）与主报告一致，由页面注入 `topBarFilters` 一并提交。

对比区使用的 **可选应用列表** 来自页面注入的 **`sidebarAppList`**（普通模式下由 `app-list` 的 `items` 提供）。

| `/daily/ad-platform`      | POST | `daily-02-ad-platform.json`      | `getAdPlatform`      |

将 `daily` 替换为 `monthly`，契约文件前缀 `monthly-00` … `monthly-08`，函数名 **相同**。

## 7. 操作场景 → 调用接口（速查）

| 场景 | 调用的接口（按大致顺序） |
| --- | --- |
| 首次进入页面（普通模式） | `GET` 顶栏 meta（见 §8）→ `{period}/app-list` → `{period}/overview` →（若当前 Tab 非汇总）对应 `02~05` 之一 |
| 切换 日报/周报/月报 | `app-list` → `overview` →（视 Tab）`02~05` |
| 改报告日期 / 改顶栏应用·平台·广告平台·国家（任一多选变化） | 同上（侧栏会重置整体，除非用户在请求中途已改选） |
| 仅切换二级 Tab | `app-list`（`tab` 变）→ `overview` →（视 Tab）`02~05` |
| 点击侧栏某一应用 | **仅** `overview` +（视 Tab）`02~05`，**无** `app-list` |
| 打开对比模式 | **停止**普通链路；并行 `compare-overview`、`compare-trends`、`compare-metrics` |
| 关闭对比模式 | `app-list` → `overview` →（视 Tab）`02~05` |
| 汇总子组件内「刷新」 | 同「点击应用」路径：仅 `refreshReportDetail`（overview + 当前 Tab） |
| 打开飞书弹窗 | `GET /lark/push-config`（若尚未加载过） |

## 9. Mock 与真实接口切换

- 文件：`src/views/product-insight/business-report/config/data-source.ts`
- 枚举：`BusinessReportReadEndpoint`（含 `DailyAppList`、`DailyOverview`、…、`LarkPushNow` 等）
- 映射：`BUSINESS_REPORT_USE_MOCK[endpoint] === true` → 走 `mockData.ts` / 本地逻辑；`false` → `src/api/business-report.ts` 真实 `request`。

**联调建议**：接真接口时至少保证 **`{period}/app-list` 与 `{period}/overview` 同时可用**，否则侧栏或 KPI 会空。
