# 登录与认证 Mock

该目录维护 `auth/login` 相关的认证契约与示例说明，当前优先覆盖以下接口：

- 登录
- 获取当前用户信息

## 当前范围

本次新增的重点是 `GET /api/v1/datacenter/biz/user/get` 的 `userInfo` 契约，用于统一前端登录后用户态、角色路由过滤、日期权限控制的基础数据结构。

## 目录说明

- `backend-api/README.md` 认证接口清单与联调说明
- `backend-api/01-user-info.json` 当前用户信息契约，覆盖角色、页面权限、日期权限、按钮权限预留结构

## 对接原则

1. `roles` 作为正式角色字段，供前端菜单、路由和权限判断使用。
2. `permissions` 暂时保留，兼容当前线上旧结构。
3. `permissionConfig` 作为新权限配置容器，承载页面权限、日期权限和按钮权限预留。
4. 日期权限语义基于“当前日期”回看：`maxHistoryDays = 30` 表示最早可选日期为今天往前 30 天，最晚可选日期为今天。
