# 关键字字典规则（查询维度字段 + 指标字段）

本字典用于统一前端 mock 契约与后端接口的“字段命名”，解决同一维度/指标在不同页面或不同接口中出现多种字段名导致的联调、验收与维护成本问题。

适用范围：本仓库 `src/main/resources/前端mock接口/**` 下的 JSON 契约，以及 `com.vegoo.datacenter.analysis` 对外接口的请求/响应 DTO 字段命名。

---

## 1. 命名总则

### 1.1 字段风格

- 接口字段风格：`camelCase`
- 说明：`t_/s_/n_/d_/b_ + snake_case` 属于数据库/数仓字段命名风格，不作为接口交互字段名
- 允许在文档中标注字段映射关系（接口字段 → 数据库字段），但接口字段本身不带类型前缀

### 1.2 同义词禁止

- 同一含义的维度/指标在所有接口中必须使用同一个字段名
- 禁止“同名不同义”（字段语义/口径变了但复用旧字段名）

### 1.3 新字段登记门禁（允许新增，但必须可追溯）

字典无法穷举未来所有字段，因此规则不是“禁止新增字段”，而是“禁止无登记的新增字段”：

- 允许在接口/契约中新增字段
- 若 mock 契约或后端 DTO 出现本字典未收录的新字段名，必须在同一个 PR 中补充本字典对应条目（字段名、类型、含义、单位/是否 Pct、口径说明）
- 未同步登记的新字段视为不合规（不得进入联调冻结点与合并阶段）

### 1.4 禁用字段名（永远不允许）

以下字段名语义高歧义或历史上易造成误解，禁止出现在接口字段中：

- `roi`、`roiPct`（必须改为 `totalRoiPct` / `roi{N}Pct` / `roiActualPct` 等明确口径）
- `revenue`（必须改为 `totalRevenue` 或明确口径的 `adRevenueReal`/`iapRevenueTotal` 等）

### 1.3 百分比字段规则（强制）

- 若字段表达“百分比数值”（如 `3.10` 表示 `3.10%`），字段名必须以 `Pct` 结尾
- `Pct` 字段数值范围通常为 `0~100`，并统一保留 2 位小数
- 若字段表达“比例”（如 `0.031` 表示 `3.1%`），字段名必须以 `Ratio` 结尾（不建议用于前端展示口径，除非有明确约定）

## 2. 查询维度字段字典（Request 常用字段）

说明：

- “标准字段”是新页面/新接口必须使用的命名
- “兼容字段”用于存量接口；新增接口禁止继续扩散兼容字段

| 分类 | 标准字段 | 类型 | 含义/口径 | 典型值 | 兼容字段（存量） |
| --- | --- | --- | --- | --- | --- |
| 时间 | `startDate` | string | 查询开始日期（含），`YYYY-MM-DD` | `2026-05-01` | `t_start_date` |
| 时间 | `endDate` | string | 查询结束日期（含），`YYYY-MM-DD` | `2026-05-31` | `t_end_date` |
| 时间 | `date` | string | 单日（用于按日查询或点选某一天），`YYYY-MM-DD` | `2026-05-05` | `t_date` |
| 时间 | `month` | string | 月（用于按月查询），`YYYY-MM` | `2026-05` | `t_month` |
| 时间 | `timeRangeDays` | int | 最近 N 天（与 `startDate/endDate` 二选一；优先使用起止日期） | `7`/`30` | `timeRange` / `n_time_range_days` |
| 应用 | `apps` | array | 应用筛选项列表（元素为 `AppPlatformItem`：`appId`+`platform`），优先级高于 `appIds` | `[{ "appId": "...", "platform": 0 }]` | — |
| 应用 | `appIds` | array | 应用 ID 列表（空/不传表示全部） | `["com.xxx.a","com.xxx.b"]` | — |
| 应用 | `appId` | string | 单个应用 ID（不建议：单值且难表达多选） | `com.xxx` | `s_app_id` |
| 平台 | `platform` | int | 平台枚举：`0=Android, 1=iOS`；空/不传表示全量 | `0` | `platformCode` / `n_platform` / `platform`（string） |
| 国家 | `countryCode` | string | 国家代码（ISO-3166-1 alpha-2），空/不传表示全部 | `US` | `s_country_code` |
| 广告平台 | `source` | int | 广告平台枚举：与数据表 `n_source` 口径一致 | `1` | `n_source` |
| 广告平台 | `sources` | array | 广告平台枚举集合（多选筛选），空/不传表示全部 | `[1,2,3]` | — |
| 广告 | `adType` | string | 广告类型（推荐传可读 key；枚举映射由后端统一处理） | `interstitial` | `n_ad_type` |
| 广告 | `adUnitId` | string | 广告单元 ID | `ca-app-pub-...` | `s_ad_unit_id` |
| 广告 | `adNetwork` | string | 广告网络名称（如 Google/Meta） | `Google` | `s_ad_network` |
| 广告 | `adNetworkType` | string | 广告网络类型（如 bidding/waterfall） | `bidding` | `s_ad_network_type` |
| 版本 | `appVersion` | string | 应用版本筛选 | `1.2.3` | `s_app_version` / `app_version` |
| 搜索 | `keyword` | string | 模糊搜索关键词 | `weather` | — |
| 分页 | `page` | int | 页码，从 1 开始 | `1` | `current` |
| 分页 | `pageSize` | int | 每页条数 | `20` | `page_size` / `size` |
| 排序 | `sortField` | string | 排序字段 key | `cost` | `sort_field` |
| 排序 | `sortAsc` | boolean | 是否升序（与 `sortField` 同时生效） | `false` | `sort_asc` |

## 4. 指标字段字典（Metric Keys）

说明：

- 小数类指标：返回时统一保留 2 位小数（HALF_UP）

- 百分比数值字段：以 `Pct` 结尾（0~100），返回前先 \*100 再保留 2 位

ROI 字段命名约定：

- 禁止使用含义不明确的 `roi`/`roiPct` 作为接口字段名
- 必须明确口径：
  - 累计 ROI：`totalRoiPct`（或比例口径 `totalRoiRatio`）
  - N 日 ROI：`roi{N}Pct`，例如 `roi1Pct`、`roi7Pct`、`roi30Pct`

### 4.5 环比/同比（变化率）

变化率属于百分比数值字段，统一使用 `Pct` 后缀：

| 字段名                  | 类型    | 含义                                     |
| ----------------------- | ------- | ---------------------------------------- |
| `changePctVsPrevPeriod` | decimal | 环比变化百分比（可为负）                 |
| `changePctVsPrevYear`   | decimal | 同比变化百分比（可为负；对比“去年同期”） |

同比（VsPrevYear）口径约定：

- 若当前查询为区间（startDate/endDate）：对比区间整体向前平移 1 年的同长度区间（例如 `2026-05-01~2026-05-31` 对比 `2025-05-01~2025-05-31`）
- 若当前查询为单日（date）：对比同一自然日的去年日期（例如 `2026-05-05` 对比 `2025-05-05`）
- 若涉及自然周/月等特殊日历对齐（如“同周”而非“同日”），必须在接口/页面的指标口径中明确说明，不得默认猜测

## 5. 兼容字段与迁移规则

### 5.1 存量字段允许存在，但必须登记

若接口已存在并且前端成本较高，允许短期保留以下存量风格字段：

- `t_start_date/t_end_date`、`s_app_id`、`s_country_code`、`n_source` 等“数据库风格字段名”
- `page_size/sort_field/sort_asc` 等 `snake_case` 字段
- 指标字段采用带前缀的数据库风格（例如 `d_cost`、`n_clicks`）

但必须遵循：

- 新增接口不得继续使用存量字段风格
- 存量接口每次大改/重构时，优先迁移到本字典的标准字段（`camelCase`）

### 5.2 迁移建议（不破坏前端）

- 优先“新增标准字段 + 保留旧字段”一段时间：
  - 后端同时返回新旧字段（旧字段标记为 deprecated，待前端迁移完成后移除）
- 破坏性变更（改名/删字段）必须先改 mock 契约并走联调冻结点
